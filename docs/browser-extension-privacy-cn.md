# Agent Memory Lab 浏览器插件：权限与隐私说明

这份说明用于本地预览、内部试用和未来 Chrome Web Store 发布前准备。当前插件默认只连接本机的 Agent Memory Lab 服务，不把网页内容上传到第三方云端。

## 一句话说明

插件会在用户浏览网页或使用 AI 产品时识别当前页面上下文，生成可审阅的记忆候选；内容进入长期记忆前必须经过本地 Viewer 审阅和确认。

## 权限说明

| 权限 | 为什么需要 | 用户可见行为 |
| --- | --- | --- |
| `activeTab` | 读取当前激活页面的标题、URL、选中文本和页面摘要 | 点击插件弹窗或侧栏时识别当前页面 |
| `tabs` | 打开本地 Viewer、识别当前标签页 | 点击“工作台”“Skill”后打开本地页面 |
| `storage` | 保存 API 地址、Viewer 地址、访问密钥和最近同步记录 | 插件设置与最近记录在本机保留 |
| `sidePanel` | 提供浏览器侧栏 | 打开“同步侧栏”查看候选记忆和经验 |
| `contextMenus` | 提供右键保存入口 | 右键页面可加入本地审阅 |
| `scripting` | 为未来按需注入能力预留 | 当前主要用于本地开发和后续站点适配 |
| `<all_urls>` content script | 允许插件在不同网页识别页面类型，并在支持的 AI 页面显示本地记忆提示 | 只有支持的 AI 页面会显示“本地记忆”提示；其他网页主要用于用户主动保存时采集上下文 |

## 默认采集内容

插件会把当前页面标准化成 `PageCapture`：

- 页面标题、URL、host、页面类型
- meta description 和可见标题结构
- 用户选中的文本
- 支持的 AI 页面中的最近可见对话片段和输入框草稿
- AI 页面诊断信息：是否找到输入框、命中的选择器、输入草稿长度和最近对话数量
- 候选记忆、候选经验、隐私风险提示

同步侧栏里的“复制诊断”只会复制页面标题、URL、域名、AI 产品识别结果、输入框是否命中、命中的 selector、草稿长度、最近对话数量，以及一个需要用户手动填写的验收模板。它用于修复不同 AI 网站的页面适配，不会主动复制 Cookie、密码、访问令牌或完整网页源码。

插件不会默认采集：

- 浏览器历史记录
- Cookie、密码、支付信息
- 页面里的隐藏表单字段
- 未打开页面的内容

## 数据流向

默认连接地址：

```text
API: http://localhost:3111
Viewer: http://localhost:3113
```

数据流：

```text
网页 / AI 输入框
  -> 浏览器插件生成 PageCapture
  -> 本地 Agent Memory Lab API
  -> Viewer 待审阅队列
  -> 用户确认后写入本地记忆或经验
```

如果本地工作台未启动，侧栏会显示“本地工作台未连接”，保存按钮会被禁用。

## 用户控制

- 保存网页需要用户点击弹窗、侧栏或右键菜单。
- 长期写入需要在 Viewer 的“待审阅”里确认。
- 审阅时可以编辑标题、内容、项目、标签和类型。
- 不合适的候选可以忽略。
- Skill 草稿只会预览和复制，不会自动写入本地 Skill 目录。
