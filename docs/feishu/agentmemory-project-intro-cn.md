# Codex Memory Lab：本地优先的 Agent 记忆工作台

一句话：把 Codex / Claude Code / Cursor 里的聊天记录、项目上下文、个人偏好、可复用经验和本地 Skill，整理成能被 Agent 继续使用的工作记忆。

## 1. 产品定位

Codex Memory Lab 解决的是长期使用 Agent 时的上下文断层：每次都要重讲项目、偏好和上次进度；做过的经验留在聊天里，没有进入可复用系统；本地 Skill 越装越多，但来源和用途不清楚。

它把 Agent 工作流拆成四层：

| 层级 | 作用 | 用户看到的入口 |
| --- | --- | --- |
| 会话层 | 保存历史对话和工具轨迹 | 会话 |
| 记忆层 | 保存稳定身份、偏好、项目背景 | 记忆 |
| 行动层 | 整理待跟进、正在做、卡住和完成事项 | 待办 |
| 能力层 | 管理本地 Skill，把成熟经验沉淀为能力 | Skill |

## 2. 当前产品截图

下面这些图来自当前本地产品，不是原项目旧图。

### 2.1 总览

总览页把最近会话、记忆、经验、状态放在一个入口里。项目卡片可以点击进入对应会话，适合作为每天打开的工作台。

[图：总览页截图]

### 2.2 会话

会话页按时间展示历史会话。用户可以看到每个项目下有多少段会话、多少条记录，以及最近更新时间。

[图：会话页截图]

### 2.3 Skill 管理台

Skill 页会扫描本机的 Codex、Agents 和插件 Skill 目录，展示每个 Skill 的来源、路径和详情。它解决的是“本地能力越来越多，但不知道从哪里来、做什么用”的问题。

[图：Skill 管理台截图]

### 2.4 待办

待办页把内部 action/frontier/priority 等概念转换成用户更容易理解的状态：待跟进、正在做、卡住、已完成。

[图：待办页截图]

## 3. 主要功能

| 页面 | 现在负责什么 | 设计重点 |
| --- | --- | --- |
| 总览 | 最近会话、记忆数量、经验数量、服务状态 | 作为第一入口，能点击跳转细节。 |
| 记忆 | 自动整理出的身份、偏好、项目线索 | 自动沉淀，但允许人工审阅、编辑、删除。 |
| 会话 | 历史会话时间线和完整过程 | 不只给摘要，保留可复盘路径。 |
| Skill | 本地 Skill 管理、搜索、详情、复制路径 | 管理本地能力，辅助经验沉淀。 |
| 待办 | 从会话中整理下一步 | 用人话状态替代内部字段。 |
| 活动 | 工作密度和活动类型观察 | 当前还需要继续优化加载体验。 |

## 4. 推荐工作流

1. 用户正常和 Codex / Claude Code / Cursor 对话。
2. 后台记录会话、工具调用、项目路径和关键上下文。
3. Viewer 把内容整理成记忆、会话、行动、经验。
4. 用户审阅重要内容，删掉不该保留的，补充缺失线索。
5. 重复出现的做法沉淀成 Skill。
6. 下次对话时，Agent 直接从上次上下文继续，而不是重新解释。

## 5. 插件与集成

| 接入方式 | 适合场景 | 作用 |
| --- | --- | --- |
| Codex 插件 | Codex CLI / Codex Desktop | 记录会话、工具调用、任务完成、上下文线索。 |
| Claude Code 插件 | Claude Code 长期项目 | 通过 hooks 和 MCP 连接记忆层。 |
| OpenCode 插件 | OpenCode 工作流 | 捕捉命令和会话片段。 |
| MCP Server | Cursor、Gemini CLI、Claude Desktop 等 | 让不同 Agent 通过 MCP 读写记忆。 |
| REST API | 自定义本地工具 | 直接接入记忆、会话、审计、搜索能力。 |

## 7. 下一步建议

| 优先级 | 任务 | 原因 |
| --- | --- | --- |
| P0 | 自动基于当前聊天记录更新记忆 | 用户多次明确要求“记忆你要自己更新”。 |
| P0 | 修复活动页加载长期停留问题 | 当前截图仍可能停留在加载态，影响产品可信度。 |
| P1 | 定期把可沉淀经验合并进 Skill | 让经验真正进入长期能力层。 |
| P1 | 明确“经验”和“结晶”的区别或合并入口 | 目前用户理解成本高。 |
| P2 | 增加飞书同步 / 一键导出 / 项目报告 | 让记忆系统能进入用户日常协作工具。 |
