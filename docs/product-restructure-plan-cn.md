## 0. 这份文档解决什么

会议把产品形态重新定向:主导航砍成 **三栏 —— 总览 / 待办 / 证据**，待办放首位、可链接证据原文；去掉与 AI 协作无关的「记忆 / Skill」主导航；待办抽取继续用关键词匹配（不接大模型）；Agent 回复用 Markdown 渲染；补主动提醒。受众锁定 **个人重度 Agent 用户**。

### 「待办」的真实定义（重要）

会议明确:**不要被「待办」字面局限**。它本质是 **Agent 主动推给用户的异步收件箱**，混着三类性质不同的条目:

1. **待回应** — Agent 运行中抛给你的问题，你正忙没空回。**Agent 在那头等着，时间敏感。** 这是后端目前最缺的语义。
2. **待处理** — 今天没做完、欠着的工作信息。对应现有 `action` 模型。
3. **已完成简报** — 今天已经做掉的事，汇报性质、不需要你动作，给的是「安心知道进展」。

投递出口:后续通过 `lark-cli + openclaw` 绑定飞书机器人，把这条收件箱追到手机上。**本期先做前端重构，飞书投递通道后做。**

### 两条独立的工作线

本方案把重构拆成两条**互不阻塞**的线:

- **线 A:前端三栏重构**（本期主线）— 以做减法 + 接线为主，后端几乎不新增。详见 §2–§7。
- **线 B:本地去 Docker**（温和，已选定）— 保留 iii-engine 与三原语、不动 AGENTS.md 铁律，只把本地启动收敛到原生二进制、去掉 Docker 这层可选重量。配置/分发层收尾，零架构改动。详见 §12。

## 2. 目标形态:三栏

```text
┌─────────────────────────────────────────────┐
│  灵感记忆台                  [总览] [待办] [证据] │
├─────────────────────────────────────────────┤
│                                             │
│   待办为第一入口；每条待办可跳转到证据原文        │
│                                             │
└─────────────────────────────────────────────┘
```

| 栏 | 定位 | 一句话 |
|---|---|---|
| **总览** | 落地页 | 今天有哪些待跟进、最近从哪些会话抽出了什么、服务状态是否正常 |
| **待办**（首位） | 产品核心 = Agent→用户收件箱 | 三分区:**待回应**（Agent 在等你）/ **待处理**（欠着的事）/ **已完成**（简报）。每条可跳证据 |
| **证据** | 待办的支撑 | 待办来自哪段会话原文；按时间线/会话重点展示原始观测，供核对 |

设计原则延续 README 已有的克制路线:少暴露内部概念（graph/audit/frontier 不进主导航）、图标优先文字克制、本地优先、自动整理但不替用户做最终决定。

### 2.1 「待办」栏的内部三分区（关键设计）

三类条目性质不同，**不能拍平成一个列表**——视觉权重、提醒强度、能否「忽略」都不同:

| 分区 | 用户该做什么 | 提醒强度 | 可忽略 | 后端来源 |
|---|---|---|---|---|
| 🔴 **待回应** | 回 Agent 一句话 | 高（Agent 在等，时间敏感） | 否（只能回应或转待处理） | ⚠️ 需新语义（见 §6） |
| 🟡 **待处理** | 自己动手做 | 中（`blocked`/`failed` 权重高） | 是 | ✅ `action`（pending/blocked） |
| 🟢 **已完成** | 知悉即可 | 无 | 自动归档 | ⚠️ `action.done`，抽取器暂不主动识别 |

默认:顶部展开「待回应」（最紧急、Agent 在等）；「已完成」折叠成一行简报（「今天完成了 N 件」）。

## 5. 专家模式:不删能力，只是收纳

会议要做减法，但后端 66 函数 / 131 接口是资产。方案不删，而是设一个**专家模式开关**（设置项或 `?expert=1`），点亮后在导航末尾出现「进阶」分组，按现有 `TAB_REDIRECTS` 名单恢复:记忆、Skill、图谱、档案、审计、回放、结晶。

这样:
- 个人重度用户默认看到干净的三栏。
- 能力没有丢失，可被重新利用，符合「最大化利用」。
- 实现成本低:这些视图渲染函数 `view-graph/profile/audit/...` 都还在 `index.html` 里，只需放开 `TAB_REDIRECTS`。

## 7. 主动提醒:难点拆解与分期

会议明确两件事:① 主动提醒有价值（5 分钟未响应弹出，可推手机/桌面）；② 优先级判定是难点——关键词抽取定不准优先级，接大模型会臃肿，本地小模型不保证每台机器都有。

分期建议:
- **P1（本期可做）**:桌面内提醒。待办列表用 `frontier`/`next` 的现有排序做「建议先做」标记，浏览器通知 API（Notification）做到「N 分钟无操作弹一条」。不引入模型，优先级用现有 `priority` + `reason` 启发式（`blocked`/`command_failed` 权重高）。
- **P2（投递通道，后做）**:跨设备推送 = `lark-cli + openclaw` 绑定**飞书机器人**，把收件箱追到手机。这是会议指定的投递方案，与本地优先有张力（消息要出本机），单独排期。本期前端不依赖它。
- **不做**:为优先级接大模型 / 强依赖本地小模型。与会议「避免臃肿」一致。

## 12. 本地去 Docker 线（温和方案，已选定）

> 决策:采用**温和方案**——保留 iii-engine 与三原语，不动 AGENTS.md 铁律，只把本地启动收敛到「原生二进制」、去掉 Docker 这层可选重量。激进的「去 engine」方案降级为附录备选（§12.5），暂不实施。

### 12.1 关键事实:Docker 早已不是必需，而是优先级最低的兜底

读 `src/cli.ts` 的 `startEngine()` 决策链（实测）:

1. PATH 上有 `iii` 二进制 + 有 config → **直接用原生二进制**，docker 完全不提及。
2. PATH 没有 → 查 `~/.local/bin` 等 `fallbackIiiPaths()`。
3. 仍没有 → **默认推荐**「安装 iii 到 `~/.local/bin`（~6MB，~5s）」，hint 标注 `recommended`。
4. Docker 只在两种情况出现:显式 `AGENTMEMORY_USE_DOCKER=1`，或原生安装失败后的兜底询问。

**结论:纯本地环境下，只要本机有 iii 二进制，整个流程一行 docker 都不碰。** Docker 在代码里就是 advanced / 失败兜底，不是主路径。

### 12.2 「本地去 docker」具体动作（零架构改动）

| 动作 | 说明 | 碰三原语 | 碰 AGENTS 铁律 |
|---|---|---|---|
| 确保本机装好原生 `iii` 二进制 | `curl …/iii-aarch64-apple-darwin.tar.gz \| tar -xz -C ~/.local/bin`（本机 darwin） | 否 | 否 |
| 启动走 `npm run start:local-memory` | 本就走原生二进制路径 | 否 | 否 |
| 个人本地版**移除/不分发 docker 物料** | `docker-compose.yml`、`iii-config.docker.yaml`、`.env.example` 的 docker 段 | 否 | 否 |
| 帮助文本弱化 docker 提示 | CLI/README 里 docker 从「可选」降为「附录」 | 否 | 否 |

iii-engine、WebSocket 总线、`StateKV → state::* → 文件 SQLite`、66 个函数注册触发——**全部毫发无损**。变的只是「engine 怎么被拉起」:从「可能走 docker」收敛成「只走本地原生二进制」。

### 12.3 收益

- 用户装一个 ~6MB 二进制，而不是拉 docker 镜像 + 维护 volume 权限（`docker-compose.yml` 里 `chown 65532` 的 init 容器复杂度消失）。
- 启动更快、无 docker daemon 依赖。
- 架构零改动，风险极低，**不阻塞也不依赖前端线**。

### 12.4 唯一待拍板:docker 物料「删除」还是「保留隐藏」

- **保留隐藏（推荐）**:代码已是「默认不用、显式才用」，本地不设 `AGENTMEMORY_USE_DOCKER=1` 即永远走原生。**几乎零改动**就满足「本地甩掉 docker」，又不堵死未来团队/服务器部署（ROADMAP Q4 长期部署）。
- **彻底删除**:个人本地版最干净，但需要一个删除 PR，并失去现成部署路径。

### 12.5 附录（备选，暂不做）:激进减重 = 去 iii-engine
